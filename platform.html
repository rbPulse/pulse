Templates
Starter packs that seed new tenants.
Template library
Save a tenant's full config (brand, terminology, workflow, catalog, comms) as a reusable starter. Apply during onboarding. Seeded from Pulse + fictional verticals as worked examples.
Phase 12
Modules
Feature modules entitled per tenant.
Module catalog
Define the module catalog (live video, ePrescribing, labs, messaging, analytics add-ons). Per-tenant entitlement comes from the plan they're on, with manual overrides allowed. Feature flags in downstream portals read from this table.
Phase 9
Integrations
Provider catalog and credential management.
Integration providers
Catalog of available providers (Stripe, Daily.co, Resend, Twilio, EHR, shipping, ID verification). Per-tenant credentials stored encrypted (Supabase Vault / pgcrypto). Webhook routing resolves incoming events back to the owning tenant.
Phase 7
Billing
Plans, subscriptions, MRR.
Pulse internal billing
Per-tenant subscription status, plan line items, monthly cost. Platform-level MRR, by-plan breakdown, churn flags. Stripe is the source of truth; operator portal embeds the Customer Portal link rather than rebuilding invoicing.
Phase 9
Analytics
Cross-tenant funnels, SLAs, capacity.
Platform analytics
Fleet dashboards plus per-tenant drill-downs (funnel drop-off, refill conversion, SLA performance, clinician load, payment failures, no-shows, inactivity). Daily snapshot job aggregates into analytics_snapshots.
Phase 14
Support
Cross-tenant support queue and implementation console.
Support queue + view-as
Unified support_requests queue across tenants (table shipped in migration 012). "View as tenant role" drops an operator into a tenant portal read-only with a persistent banner. Diagnostic checks per tenant (integration health, queue depth, stale workflows, expired licenses).
Phase 13
Audit Log
Config changes and platform-originated actions.
Change management
Every config change logged with before/after snapshot. Cross-tenant filter for "what did Pulse internal change." Per-tenant timeline lives on the tenant detail page.
Phase 10
Settings
Platform-level configuration and operator roster.
Platform settings
Operator directory (users carrying a platform_role, with role assignment and revocation). Platform-level defaults. Global feature flags. Reserved for anything that isn't tenant-scoped.
